# Политика безопасности

## Поддерживаемые версии

Исправления безопасности выпускаются для последней стабильной версии 4phone
Windows. Предварительные beta-версии поддерживаются только до выхода следующей
beta или stable-версии.

## Сообщение об уязвимости

Не публикуйте сведения об уязвимости в обычной GitHub issue. Используйте
приватное сообщение через раздел Security Advisories репозитория:

https://github.com/XANDER-CAGE/4phone-windows-client/security/advisories/new

Укажите затронутую версию, способ воспроизведения, влияние и, если возможно,
предлагаемое исправление. Не включайте реальные пароли, токены, SIP-учетные
данные или персональные данные.

Мы постараемся подтвердить получение сообщения в течение 3 рабочих дней и
сообщить план исправления после первичной проверки. Срок публикации согласуется
с исследователем и зависит от критичности.

## Проверка релизов

Стабильный установщик должен:

- иметь действительную Authenticode-подпись с timestamp;
- содержать исполняемый файл с отдельной действительной Authenticode-подписью;
- иметь корректную Ed25519-подпись в appcast;
- совпадать с опубликованной SHA-256 суммой;
- сопровождаться SBOM и provenance.

Неподписанные beta-сборки явно помечаются как `unsigned-beta` и не предлагаются
автоматическим каналом stable.
