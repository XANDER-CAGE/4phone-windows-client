# Конфиденциальность 4phone Windows

Дата вступления в силу: 14 июля 2026 года.

4phone Windows подключает пользователя к облачной телефонии 4phone. Клиент не
содержит рекламной аналитики и не отправляет самостоятельную телеметрию.

## Какие данные обрабатываются

- Email, пароль и код двухфакторной аутентификации передаются только в
  `https://api.4phone.uz` для входа. Пароль и код не сохраняются клиентом.
- Access token хранится только в памяти процесса. Refresh token хранится в
  Windows Credential Manager для текущего пользователя.
- SIP-логин, домен и защищенный Windows DPAPI SIP-пароль сохраняются в профиле
  текущего пользователя для регистрации телефона.
- Журнал звонков, контакты, настройки, диагностические журналы и выбранные
  пользователем записи звонков хранятся локально на компьютере.
- При проверке обновлений клиент запрашивает appcast у `api.4phone.uz`, а
  установщик скачивается с GitHub Releases.

## Передача данных

Сигнализация звонков передается инфраструктуре 4phone и настроенному SIP-узлу.
Медиа передается согласно конфигурации телефонии. Данные могут обрабатываться
GitHub только при скачивании публичного обновления.

## Управление и удаление

Выход из учетной записи отзывает refresh token и удаляет локальные учетные
данные 4phone. Удаление приложения сохраняет пользовательские настройки, чтобы
не потерять контакты и историю. Их можно удалить вручную из:

- `%APPDATA%\4phone`
- `%LOCALAPPDATA%\4phone`

Локальные записи звонков удаляются пользователем из выбранного им каталога.

## Безопасность

API используется только по HTTPS с системной проверкой сертификата. SIP-пароль
защищается DPAPI. Установщики проверяются Authenticode и подписью обновления
Ed25519 в соответствии с `CODE_SIGNING_POLICY.md`.

Вопросы о данных и запросы субъекта данных направляйте через контактную форму
на https://4phone.uz/.
