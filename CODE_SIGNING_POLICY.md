# Политика подписи релизов

## Область действия

Политика применяется к публичным релизам 4phone Windows. Authenticode-подпись
предоставляется SignPath Foundation после одобрения проекта. В свойствах файла
издателем может отображаться SignPath Foundation.

## Ключи и доступ

- Закрытый Authenticode-ключ не хранится в репозитории или GitHub Secrets.
  Подпись выполняется изолированным сервисом SignPath.
- Закрытый Ed25519-ключ WinSparkle хранится только в защищенном GitHub
  environment `release-signing`.
- Публичный Ed25519-ключ закреплен в исходном коде клиента.
- Доступ к выпуску требует MFA и ручного подтверждения environment.
- Секреты не передаются workflow из pull request или fork.

## Последовательность выпуска

1. Воспроизводимо собрать неподписанный `4phone.exe`.
2. Передать хеш и артефакт EXE в SignPath.
3. Проверить Authenticode-подпись EXE и timestamp.
4. Собрать Inno Setup installer только с подписанным EXE.
5. Передать установщик в SignPath.
6. Проверить Authenticode-подпись установщика и timestamp.
7. Подписать финальный установщик ключом Ed25519 WinSparkle.
8. Создать SHA-256, SBOM и provenance для финального байтового содержимого.
9. Опубликовать GitHub Release и подписанный appcast.

Изменение любого подписанного файла после соответствующего шага запрещено.

## Требования stable-канала

Stable-релиз блокируется, если:

- подпись EXE или установщика отсутствует либо недействительна;
- отсутствует доверенный timestamp;
- Ed25519-подпись не проходит проверку;
- версия appcast не совпадает с числовой версией ресурсов;
- SHA-256, SBOM или provenance отсутствуют;
- исходный commit не принадлежит защищенной ветке;
- ручное подтверждение release-signing не получено.

До одобрения SignPath допускается только явно помеченный неподписанный
`unsigned-beta`. Он не публикуется в stable appcast.

## Отзыв и реагирование

При компрометации ключа публикация прекращается, секрет отзывается, публичный
ключ обновляется отдельным проверяемым релизом, а затронутые версии удаляются из
appcast. Инцидент документируется в GitHub Security Advisory.
